# Day 82 - Morse Code Converter

## Overview
A text-based command line program that converts any string input to and from Morse Code. Supports letters, numbers, punctuation, and special characters.

## Features

- **Text to Morse Code**: Convert alphanumeric text into dots and dashes
- **Morse Code to Text**: Decode Morse Code back to readable text
- **Full Character Set**: Supports A-Z, 0-9, and common punctuation (.,?!()&:;=+-_\"$@)
- **Word Separation**: Uses `/` between words in Morse Code output
- **Interactive Menu**: Clean, navigable terminal interface
- **Error Handling**: Validates inputs and handles unexpected errors gracefully

## Usage

```powershell
python Day082/morse_code.py
```

## How It Works

1. **Dictionary Mapping**: Characters are mapped to their Morse Code equivalent using a Python dictionary
2. **Reverse Mapping**: `MORSE_TO_TEXT` is generated by inverting the main dictionary for decoding
3. **Text → Morse**: Each character is converted to uppercase, looked up in the dictionary, and joined with spaces
4. **Morse → Text**: Input is split by ` / ` (word boundaries) and spaces (letter boundaries), then decoded character by character

## Key Concepts

- **Dictionary Data Structures**: Bidirectional character mapping
- **String Manipulation**: `split()`, `join()`, `upper()` methods
- **List Comprehensions**: Generating the reverse lookup table
- **Input Validation**: `try/except` blocks for error handling
- **Terminal UI**: `os.system()` for cross-platform screen clearing

## Reflection

### How I Approached the Project

I started by reviewing the repository's existing projects (Days 1-10) to understand the conventions — things like file naming, the `if __name__ == "__main__"` guard pattern, `clear_screen()` helpers, and the `try/except` error handling style. Once I had a mental model, I built the core conversion logic first (the dictionary and two translation functions), then wrapped it in the interactive menu loop, and finally added the README.

### What Was Hard

The Morse-to-text decoding was trickier than expected. Morse doesn't inherently signal word boundaries, so I had to pick a convention — I went with ` / ` (space-slash-space) for word separation. Handling edge cases like extra whitespace in user input required a `strip()` call and careful split logic. The ASCII art logo also threw a Python `SyntaxWarning` about an invalid escape sequence until I made it a raw string with `r'''`.

### What Was Easy

The text-to-Morse direction was straightforward — it's essentially a dictionary lookup per character with a `.join()` at the end. The overall project structure was familiar from the earlier days, so I didn't have to think much about scaffolding.

### How I Might Improve for the Next Project

I could add unit tests upfront rather than relying on ad-hoc terminal verification. Writing tests for the pure conversion functions would have caught edge cases sooner and given me confidence during refactoring.

### Biggest Learning

Generating `MORSE_TO_TEXT` by inverting `MORSE_CODE` with a dictionary comprehension (`{v: k for k, v in MORSE_CODE.items()}`) was a clean trick. Previously I might have manually written out a second dictionary, which would be a maintenance headache. The principle of "single source of truth" for data is something I'll carry forward.

### What I'd Do Differently

If I tackled this again, I'd:

1. **Add a test file** (`test_morse_code.py`) with pytest before writing the UI layer
2. **Support international characters** — the current version only handles ASCII, but Morse Code has extensions for accented letters (e.g., `ä` is `.-.-`)
3. **Add a learn/practice mode** where the program quizzes you on random characters in Morse
4. **Save output to file** — the ability to export conversions as `.txt` or `.wav` (audio) would make it more practical
5. **Use a `match/case` statement** (Python 3.10+) instead of `if/elif` for the menu choices for cleaner code
